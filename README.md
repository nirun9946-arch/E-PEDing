# E-peding for Nation

เว็บรวมประกาศจัดซื้อจัดจ้างภาครัฐจากระบบ **e-GP (กรมบัญชีกลาง)**
คัดเฉพาะงานที่ **เนชั่น** มีศักยภาพเข้าประมูล — ประชาสัมพันธ์ ผลิตสื่อ/รายการ
โทรทัศน์ ออกอากาศ โฆษณา อีเวนต์ คอนเทนต์ออนไลน์

ข้อมูลอัปเดตอัตโนมัติวันละ 3 รอบ (09:30, 14:00, 16:30 น.) ผ่าน GitHub Actions

## เว็บออนไลน์
https://nirun9946-arch.github.io/E-PEDing/

## ปุ่มบนหน้าเว็บ
- **🔄 ดึงข้อมูลล่าสุด** — โหลดข้อมูลชุดใหม่ที่ระบบเก็บไว้มาแสดงทันที ไม่ต้องรีเฟรชหน้า ใครก็กดได้
- **สั่งดึงจาก e-GP รอบใหม่** — พาไปหน้า Actions เพื่อกด Run workflow ให้ระบบไปดึงประกาศชุดใหม่จาก e-GP
  ใช้เวลาราว 4-5 นาที เสร็จแล้วค่อยกดปุ่มแรก (ต้องเป็นเจ้าของ repo ถึงกดรันได้)

> หน้าเว็บเรียก e-GP ตรง ๆ ไม่ได้ เพราะ e-GP ไม่เปิด CORS ให้เว็บอื่นเรียกข้ามโดเมน
> ตัวดึงข้อมูลจริงจึงต้องรันฝั่งเซิร์ฟเวอร์ (GitHub Actions) แล้วเก็บผลไว้ใน `data.js`

## รันในเครื่อง
ดับเบิลคลิก **run.bat** → รอดึงข้อมูล 3-8 นาที → เปิด `index.html`

หรือ
```
python egp_fetcher.py --days 21    # ย้อนหลัง 21 วัน
python egp_fetcher.py --days 7     # เร็วกว่า
```

## มันทำงานยังไง
1. `egp_fetcher.py` ยิงค้นหาไปที่ e-GP (`process3.gprocurement.go.th` servlet FPRO9965)
   ด้วยคำค้นสายสื่อ 16 คำ × ประกาศ 2 ประเภท:
   **ร่าง TOR** (กำลังจะเปิด) และ **ประกาศเชิญชวน** (ยื่นซองได้เลย)
2. รวมผล ตัดซ้ำ แล้วให้คะแนนความเหมาะสมกับธุรกิจเนชั่น
   คำบวก เช่น ผลิตสื่อ/สกู๊ป/ออกอากาศ · คำลบ เช่น ก่อสร้าง/ครุภัณฑ์/ป้าย/งานซื้อของ
3. เขียนผลลง `data.json` + `data.js` ให้ `index.html` แสดงผล

ปรับคำค้นและน้ำหนักคะแนนได้ที่ตัวแปร `KEYWORDS`, `POSITIVE`, `NEGATIVE` ใน `egp_fetcher.py`

## การอัปเดตอัตโนมัติ
`.github/workflows/daily-fetch.yml` รันวันละ 3 รอบ (09:30, 14:00, 16:30 น. เวลาไทย)
แล้ว commit ข้อมูลใหม่กลับเข้า repo เว็บจึงอัปเดตเอง ไม่ต้องแตะอะไร

- กดรันเองได้ที่แท็บ **Actions → ดึงข้อมูล e-GP ทุกวัน → Run workflow**
- ถ้าวันไหน e-GP ไม่ตอบสนอง job จะข้ามการ commit และคงข้อมูลเดิมไว้ (ไม่ทับด้วยหน้าว่าง)
- ต้องตั้ง **Settings → Actions → General → Workflow permissions = Read and write permissions**
  ไม่งั้น job จะดึงข้อมูลได้แต่ commit กลับไม่ได้
- ถ้า repo เงียบนานหลายเดือน GitHub อาจพักตารางเวลาอัตโนมัติไว้ก่อน แล้วส่งอีเมลแจ้ง
  กด Run workflow หนึ่งครั้งก็กลับมาทำงานต่อ
